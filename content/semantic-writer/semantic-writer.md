# Semantic Content Writer

Write or rewrite content following Koray Tugberk Gubur's 41 Authorship Rules ("Koraynese"). This produces content that is dense with entity-attribute pairs, structured with Question H2s and 40-word extractive answers, avoids all fluff, and follows strict sentence-level semantic precision. Content written this way is immediately recognizable: no hedging, no vague claims, every sentence carries a semantic payload.

---

## The 41 Authorship Rules

### Sentence Structure (Rules 1–8)

**1. Subject-First Priority**
Put the ranking target entity in the subject position. NLP parsers assign higher weight to early tokens. Whatever sits in the subject position becomes the most "salient" element.

- Good: "Technical SEO audits identify crawl errors that prevent indexing."
- Bad: "Crawl errors that prevent indexing are identified by technical SEO audits."

Both sentences mean the same thing, but relevance changes completely based on what's in the subject position.

**2. Semantic Triples**
Every sentence follows Subject-Predicate-Object (SPO) or Entity-Attribute-Value (EAV) structure. This helps search engines extract facts efficiently.

- SPO: "Google [subject] uses [predicate] semantic understanding [object] to rank content."
- EAV: "Core Web Vitals [entity] threshold [attribute] is 2.5 seconds for LCP [value]."

**3. Short Dependency Trees**
Use short sentences. One piece of information per sentence. Long sentences dilute the semantic relationship between words. Google translates short sentences more accurately — the same principle applies to NLP parsing.

- Good: "Page speed affects rankings. Google measures three Core Web Vitals. LCP must load within 2.5 seconds."
- Bad: "Page speed, which is measured through Core Web Vitals including LCP that should load within 2.5 seconds, is an important factor that affects how Google ranks websites."

**4. Active Voice**
Active voice makes the subject entity more salient. Passive voice shifts salience to the object. Use active voice unless you intentionally want to emphasize a different entity.

**5. Ranking Target in Subject Position**
The entity you want to rank for goes in the subject of sentences. This is the practical application of salience.

**6. First-Sentence Responsiveness**
The first sentence after a heading is the most important for predictive Information Retrieval. It must directly respond to the heading's question. Never delay the answer.

**7. Condition Placement**
Place if/because conditions in the SECOND part of sentences. Lead with the declaration.

- Good: "Schema markup improves click-through rates because rich snippets increase SERP visibility."
- Bad: "Because rich snippets increase SERP visibility, schema markup improves click-through rates."

**8. Verb-Led Lists**
How-to list items start with a verb/instruction.

- Good: "Run a site crawl. Identify broken links. Fix redirect chains."
- Bad: "A site crawl should be run. Broken links need identification."

### Certainty & Precision (Rules 9–14)

**9. No Opinions**
Only verifiable facts. Search engines reward E-E-A-T signals. If a claim can't be verified, don't make it.

**10. No Hedging Language**
NEVER use "can", "might", "should", "may", "could", "possibly", "generally", "tends to" when stating facts. Use present-tense fact declarations.

- Good: "Internal linking passes PageRank between pages."
- Bad: "Internal linking can help pass PageRank between pages."

**11. Numeric Precision**
Use exact numbers. "3 methods" not "several methods." "47% increase" not "significant increase." Numbers signal factual precision to ranking algorithms.

**12. Source Authority**
Reference a source or authority before making a statement. This builds E-E-A-T signals.

**13. Zero Fluff**
Every word must add semantic value. Cut filler words, unnecessary adjectives, and transitional fluff. If removing a word doesn't change the meaning, remove it.

**14. Minimum Viable Length**
Content should be as short as possible and as long as necessary. Don't pad for word count. Don't truncate to seem concise. Cover the topic completely, then stop.

### Entity & Context (Rules 15–20)

**15. Entities Over Keywords**
Focus on entities and their attributes, not keyword density. Cover the entity's full attribute map.

**16. Complete EAV Coverage**
Include all relevant Entity-Attribute-Value triplets per topic. Completeness signals topical authority.

**17. Entity Qualification**
Every entity mention gets descriptive context. Don't name-drop — qualify.

- Good: "Koray Tugberk Gubur, the founder of Holistic SEO & Digital and creator of the semantic SEO framework, developed 41 authorship rules."
- Bad: "Koray developed 41 rules."

**18. Entity Disambiguation**
Avoid coreference errors. When multiple entities share gender or type, use full names, not pronouns.

- Bad: "She reviewed his framework and she agreed." (Who is "she"?)
- Good: "Sarah Teach reviewed Koray's framework. Teach confirmed the methodology's effectiveness."

**19. Contextual Predicates**
Use industry-specific verbs. Context-specific predicates increase relevance.

- Legal: "filing", "adjudicating", "litigating" (not "doing", "making")
- SEO: "crawling", "indexing", "rendering" (not "checking", "looking at")

**20. NER Coverage**
Include all relevant named entities with proper recognition signals. Each entity should be identifiable by NLP systems.

### Answer Format (Rules 21–24)

**21. 40-Word Extractive Answers**
Immediately after each Question H2, provide a concise ~40-word answer. This is the extractive answer target for featured snippets and AI overviews.

**22. Direct Answer Placement**
First sentence after question heading = definitive answer. Never start with context, background, or "Well, it depends." Answer first, then elaborate.

**23. Featured Snippet Format**
Target answers: < 340 characters, ~40 words. Structured as a direct, complete answer to the heading question.

**24. Lists and Tables**
Use lists for enumerations and steps. Use tables for comparisons and data. These formats are preferred by featured snippets and AI extraction.

### Tone & Style (Rules 25–30)

**25. No Everyday Language**
Write with professional precision. Avoid casual phrasing.

**26. No Analogies**
Analogies dilute semantic precision. State facts directly.

- Bad: "Think of internal linking like roads connecting cities."
- Good: "Internal links distribute PageRank and signal topical relationships between pages."

**27. No Negative Content**
Avoid negative sentences, hateful words, or stressful scenarios. Focus on solutions and outcomes.

**28. Heartfelt SEO**
Write with deep empathy for the reader. Understanding the user's situation and addressing their actual needs improves engagement signals and real-world interaction metrics.

**29. Clear Structure**
Every section has a clear purpose. The reader should never wonder "why am I reading this?"

**30. No Complex Metaphors**
Literal, precise language. Metaphors create ambiguity for NLP parsers.

### Structure (Rules 31–36)

**31. One Macro Context Per Page**
67–75% of the page focuses on ONE primary theme. Never split focus between multiple macro topics.

**32. Supplementary Context**
25–33% covers related questions, commercial bridges, and FAQ sections. Supports the macro context.

**33. Question H2s**
All H2 headings formatted as questions users actually ask.

**34. Strategic Heading Order**
The heading vector (order of headings) determines the perceived context of each passage. Order headings to match the most natural information progression.

**35. Lists for Steps**
Any sequential process = numbered list. Any collection = bulleted list.

**36. Tables for Comparisons**
Any comparison between 2+ items = table format.

### Lexical & Semantic (Rules 37–41)

**37. N-Gram Consistency**
Maintain consistent 2-gram to 6-gram patterns throughout the page that signal the main theme. Repeat key concept phrases naturally.

**38. Full Lexical Coverage**
Include synonyms, antonyms, hyponyms, meronyms, collocations, and troponyms related to the central entity. Lexical depth = authority signal.

**39. Derivational Relations**
Use morphological variants: "optimize" → "optimization", "optimized", "optimizer", "optimizing."

**40. Jargon Mapping**
When using technical terms, include plain-language equivalents for accessibility and broader query matching.

**41. Associative Relations**
Include contextually linked concepts that aren't taxonomically related but frequently co-occur with the topic.

## Writing Process

### When writing NEW content:
1. If a content brief exists, follow it exactly for structure, entities, and link plan
2. If no brief exists, first identify: macro context, central entity, target query
3. Structure all H2s as questions
4. Write 40-word extractive answers immediately after each H2
5. Apply subject-first sentence structure throughout
6. Ensure every sentence follows SPO/EAV pattern
7. Remove all hedging, fluff, opinions, and analogies
8. Check entity disambiguation
9. Verify lexical coverage across all 17 relationship types
10. Validate the 67/33 macro/supplementary split

### When REWRITING existing content:
1. Identify the macro context and central entity
2. Flag all hedging language, opinions, fluff, and analogies
3. Restructure sentences: subject-first, short dependency trees
4. Convert headings to question format
5. Add 40-word extractive answers after each H2
6. Fill entity gaps (missing EAV triplets)
7. Add missing lexical relationships
8. Ensure 67/33 macro/supplementary split
9. Recommend internal links with anchor text

## Output

When writing or rewriting, deliver:
1. **The fully written/rewritten content** following all 41 rules
2. **Entity coverage checklist** — which entities and EAV triplets are included
3. **Lexical coverage notes** — which relationship types are represented
4. **Internal linking recommendations** — suggested anchor texts and target pages
5. **Schema recommendations** — which structured data to implement
